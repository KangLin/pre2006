capCaptureSequenceNoFile

康  林

你能用此宏或者明确的发送 WM_CAP_SEQUENCE_NOFILE 消息，开始一个捕获不写入文件的视频流。

BOOL capCaptureSequenceNoFile(
  hwnd  
);

参数：

hwnd
捕获窗口的句柄

返回值：

成功返回 TRUE，否则返回 FALSE。

注释：

这个宏能有效的使回调函数与视频流和音频流相关联，能让你的应用程序直接处理视频和音频数据。

如果你想改变参数控制捕获流，那应该在捕获开始前用 capCaptureSetSetup 宏设置参数。

默认在一个捕获窗口正在捕获期间，不允许其它的应用程序运行。要强制运行的话，就读设置 CAPTUREPARMS 结构的成员 fYield 为 TRUE 或者安装一个产生(yield)回调函数。

在捕获流期间，捕获窗口能明确地向你的应用程序传送指明类型的（specific types of conditions）通知。要为这些通知安装回调函数，用下面的宏：

capSetCallbackOnError
capSetCallbackOnStatus
capSetCallbackOnVideoStream
capSetCallbackOnWaveStream
capSetCallbackOnYield

QuickInfo
