排除是外接DLL导入引起的错误。
排除由使代码调整到屏幕中心语句“tp.TryToShow(vsPaneShowHow.vsPaneShowCentered) '调整到屏幕中心”引起的错误。

2、DLL导入函数输出字符串不正确。
由DLL 导入函数输出的字符串不能正确得到长度。是因为 VB 用的是 BSTR 字符串，在传给 DLL 函数前，先转换 BSTR 为 ABSTR ，然后再传给 DLL 函数。DLL 函数处理完后，以 LPTSTR 传回。所以导入函数得不到字符串的长度。
解决方法：先用一个字符串 szStr 调用 DLL 函数。如果 DLL 函数返回得到字符的长度，那么用 szOut = Left(szStr, iLen)。

以上的解决方法不完全正确。在有汉字的情况下会出现错误。完整解决方案如下：

在 API 函数定义中用 BSTR 。例如：
extern "C" long WINAPI FormatVCCodeW( /*in*/CComBSTR* lpbszInCode, /*out*/CComBSTR* lpbszOutCode);
在 VB 调用声明中用 UNICODE 调用。例如：
Public Declare Unicode Function FormatVCCode Lib "Syntax" Alias "FormatVCCodeW" (ByRef szInCode As String, ByRef szOutCode As String) As Integer
注意：以上都是用的地址传递。如果值传递，则 CComBSTR 在析构时出现错误。产生错误的原因可能是：VB 是用的 BSTR ，而 API 用的是 CComBSTR ，所以 VB 不能调用 CComBSTR 的拷贝构造函数，而只是附加上的。(2005年7月29日)
